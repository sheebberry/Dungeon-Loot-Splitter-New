Reflection Assignment

State is saved by putting lootItems and partySize into one object and storing it in localStorage whenever the data changes. When the page loads, the program reads that object from localStorage, checks that the data is valid, and restores it back into the variables. The saved party size is also placed back into the input field so the user can see it.

updateUI() is called after restoring the data because loading the variables alone does not update the screen. The lootItems array could exist in memory, but the page would still appear empty until the interface is rebuilt. Calling updateUI() makes the restored data visible.

JSON.stringify() converts a JavaScript object into text so it can be stored in localStorage, which only stores strings. JSON.parse() converts that text back into a JavaScript object the program can use.

While testing with a breakpoint inside restoreState(), I paused the code after JSON.parse() and checked the parsed object in the browser developer tools. I could see the saved lootItems and partySize, confirming the data was still there after the page reload.

This phase connects to the course outcome of building programs where data controls the interface and state is managed separately from the UI.
